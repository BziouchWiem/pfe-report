\section{Introduction}
Ce sprint est consacré à la gestion des projets et aux intégrations nécessaires pour connecter InvisiThreat à l'écosystème existant. L'objectif est de permettre la création de projets, la configuration des dépôts et l'automatisation des déclenchements via OAuth et clés API.

\section{Objectifs du sprint et planification des tâches}
\subsection{Objectifs du sprint}
\begin{itemize}
  \item permettre la création et la configuration des projets ;
  \item intégrer les dépôts GitHub via OAuth ;
  \item gérer les clés API pour les pipelines CI/CD ;
  \item initialiser des notifications de base sur les scans.
\end{itemize}

\section{Technologies et concepts utilisés}
L'intégration GitHub s'appuie sur le flux OAuth2 afin d'autoriser l'accès aux dépôts sans exposer d'identifiants. Les clés API sont générées et stockées de manière sécurisée pour authentifier les appels automatisés. Le modèle de projet regroupe dépôts, URLs cibles et paramètres de scan afin de centraliser la configuration.

\section{Réalisation}
\subsection{Gestion des projets et des dépôts}
Le module projet centralise les métadonnées (nom, dépôt, URLs, paramètres) et sert de point d'entrée pour les scans. La configuration est persistée afin d'assurer la traçabilité et l'historisation.

\subsection{Intégrations OAuth et clés API}
L'intégration OAuth permet d'accéder aux dépôts GitHub pour récupérer les métadonnées et autoriser les scans. Les clés API offrent un mode d'authentification technique pour les pipelines CI/CD, avec rotation et révocation possibles.

\section{Test et validation}
\subsection{Tests réalisés}
Les tests incluent la création de projets, la connexion OAuth, la génération de clés API et la vérification des droits d'accès selon le rôle.

\subsection{Validation}
La validation s'appuie sur des scénarios CI/CD simulés déclenchant des scans et vérifiant la récupération correcte des projets et dépôts.

\section{Conclusion}
Le Sprint 2 a permis d'ouvrir la plateforme aux intégrations essentielles. La gestion des projets, des dépôts et des clés API prépare le terrain pour les sprints orientés scanners et orchestration.
